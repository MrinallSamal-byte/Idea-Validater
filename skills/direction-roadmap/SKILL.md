# Direction & Roadmap

Most roadmaps are a build list in disguise — they assume the idea works and
sequence features. This one is built around *learning*: do the cheapest thing
that could prove the idea wrong, before sinking time and money into it.

## Method

1. **Name the riskiest assumption.** Pull it from a prior validate-idea or
   pre-mortem if available, or identify it now: the one belief that, if false,
   makes everything else pointless. Sequence everything around testing it.

2. **Define the cheapest decisive test.** For that assumption, what's the smallest,
   fastest, cheapest thing that would give a real signal? Prefer talking to 10
   real potential users, a fake-door landing page, a manual "concierge" version,
   or a throwaway prototype over building the actual product. State the test, the
   cost/time, and — critically — the result that would make the user *stop*.

3. **Stage the path.** Lay out phases where each one unlocks the next and each has
   a clear go/no-go gate:
   - **Stage 0 — Validate the riskiest assumption** (the test above).
   - **Stage 1 — Prove people will use/pay** (smallest usable version, real users).
   - **Stage 2 — Prove it can grow** (one repeatable channel).
   - **Stage 3 — Prove it can scale / sustain** (economics, ops, or for creative work, cadence).
   Tailor the stages to the idea type; don't force a startup frame onto a creative project.

4. **Make it concrete.** For each stage: the goal, the 2–4 actions, the success
   metric, and the explicit signal that means "stop or pivot." Vague roadmaps
   ("get users, grow, monetize") are useless — name the actual first three things
   to do this week.

5. **Flag what to NOT do yet.** Premature work is a silent killer. Name the
   tempting things (raising money, hiring, polishing, scaling) that should wait
   until an earlier stage's gate is passed.

## Output structure

```
DIRECTION: <idea>

RIGHT NOW, THE RISKIEST ASSUMPTION IS
One sentence. Everything below is about testing it cheaply.

THE FIRST TEST (this week)
What to do, what it costs, how long, and the result that would stop you.

THE STAGED PATH
Stage 0–3. Each: goal · key actions · success metric · go/no-go gate.

DO THESE 3 THINGS FIRST
The literal next three actions, concrete enough to start today.

DON'T DO THIS YET
Tempting work to defer, and the gate that unlocks it.
```

Stay honest here too: if the idea hasn't been validated, say the roadmap is
provisional and the real first step is validation, not building. Offer
**validate-idea** or **market-scan** if those haven't been run.
